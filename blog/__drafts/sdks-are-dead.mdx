## The Death of SDKs

In the age of Agentic AI, traditional Software Development Kits (SDKs) are becoming increasingly obsolete. SDKs, which provide a set of tools and libraries for building applications, are rigid and require extensive coding to handle various scenarios. As Brian Casel demonstrated, automations built with SDKs or no-code platforms like [n8n](https://n8n.io/) can become complex, brittle, and hard to maintain.

Agentic AI with Claude Code Skills, on the other hand, allows developers to define high-level goals and constraints, letting the AI reason and adapt dynamically. This shift from static (deterministic) code to intelligent reasoning means that applications can evolve and respond to new challenges without the need for constant manual updates.

## Embracing Agentic AI with Claude Code Skills

To leverage the power of Agentic AI, developers should focus on:

1. **Goal-Oriented Design**: Define what the application should achieve rather than how to achieve it step-by-step. This allows the AI to determine the best course of action.
2. **Dynamic Adaptation**: Utilize AI's ability to adapt to new inputs and scenarios, reducing the need for hard-coded logic and extensive conditionals.
3. **Continuous Learning**: Implement feedback loops where the AI can learn from its actions and improve over time, enhancing the application's performance and reliability.
4. **Integration with AI Services**: Leverage existing AI platforms and services that offer agentic capabilities, reducing the need to build complex systems from scratch.
5. **Focus on User Experience**: With AI handling the complexity, developers can concentrate on creating intuitive and engaging user experiences.

### AI Services Examples

* **Claude Code Skills**: As demonstrated by Brian Casel, Claude Code Skills allow for building flexible automations that can adapt to changing requirements without extensive rewrites.
* **Code Copilots**: An AI-powered coding assistant that helps developers write code faster and with fewer errors.
  * **Custom AI Agents**: Tailored AI agents that can perform specific tasks or workflows, adapting to user needs dynamically. Examples:
    * The [HAPI MCP Visual Identity System Agent](https://docs.mcp.com.ai/agents/vs-code/visual-identity-system.agent.md), which helps maintain and enhance visual identity systems. Based on the Brian's example, users can create visual assets dynamically without rigid templates.
    * [VS Code AI Agents](https://code.visualstudio.com/docs/copilot/customization/custom-agents) that assist developers in code generation, debugging, and optimization.
* **AI-Powered APIs, a.k.a. MCP Servers**: APIs that leverage AI to provide intelligent responses and actions, reducing the need for complex SDKs.

## The Future is Agentic

The future of application development lies in embracing Agentic AI. As traditional SDKs become less relevant, any person building software (developers or non-developers) must adapt to this new paradigm by focusing on high-level goals, dynamic adaptation, and continuous learning. By doing so, they can build smarter, more resilient applications that can evolve with changing needs and challenges.

Brian Casel's experience is a clear indicator that the era of rigid SDKs is coming to an end. The future belongs to those who can harness the power of Agentic AI to create flexible, intelligent systems that can think and adapt on their own.
